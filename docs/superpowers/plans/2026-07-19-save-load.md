# World Save and Load System Implementation Plan

This document outlines the step-by-step implementation plan for adding a world save and load system using Serde, Bincode, and Zlib compression.

---

### Task 1: Add Dependencies & Derive Serialization for Core Enums

**Files:**
- Modify: `Cargo.toml`
- Modify: `src/world.rs`
- Modify: `src/inventory.rs`
- Modify: `src/player.rs`

- [ ] **Step 1: Add serialization dependencies to Cargo.toml**
  Add the following lines under `[dependencies]` in `Cargo.toml`:
  ```toml
  serde = { version = "1.0", features = ["derive"] }
  bincode = "1.3"
  flate2 = "1.0"
  ```

- [ ] **Step 2: Derive Serialize/Deserialize on game enums**
  - In `src/world.rs`, derive serialization for `BlockType`:
    ```rust
    #[derive(Debug, Clone, Copy, PartialEq, Eq, Hash, serde::Serialize, serde::Deserialize)]
    pub enum BlockType { ... }
    ```
  - In `src/inventory.rs`, derive serialization for `Item` and `GameMode`:
    ```rust
    #[derive(Debug, Clone, Copy, PartialEq, Eq, Hash, serde::Serialize, serde::Deserialize)]
    pub enum Item { ... }

    #[derive(Debug, Clone, Copy, PartialEq, Eq, serde::Serialize, serde::Deserialize)]
    pub enum GameMode { ... }
    ```
  - In `src/player.rs`, derive serialization for `DamageSource`:
    ```rust
    #[derive(Debug, Clone, Copy, PartialEq, Eq, serde::Serialize, serde::Deserialize)]
    pub enum DamageSource { ... }
    ```

- [ ] **Step 3: Verify basic compilation**
  Run: `cargo check`

### Task 3: Implement SaveManager with Region Cache

**Files:**
- Modify: `src/save.rs`

- [ ] **Step 1: Implement SaveManager constructor and folders setup**
  Implement `SaveManager::new` which ensures `saves/world_001/regions/` folder exists.
  ```rust
  pub struct SaveManager {
      pub world_dir: std::path::PathBuf,
      region_cache: std::collections::HashMap<(i32, i32), RegionData>,
  }
  ```

- [ ] **Step 2: Implement Chunk compression & decompression**
  Implement helpers using `flate2::write::ZlibEncoder` and `flate2::read::ZlibDecoder` to compress/decompress block and light arrays of a chunk.

- [ ] **Step 3: Implement Region file read/write**
  Implement `SaveManager::load_chunk(cx, cz)` and `SaveManager::save_chunk(cx, cz, chunk_data)`. The manager will load the region `r.rx.rz.bin` file into cache, perform mutations/lookups on the region's hashmap, and flush region files back to disk when saved.

- [ ] **Step 4: Implement Player & Level file I/O**
  Implement `SaveManager::save_player_and_level` and `SaveManager::load_player_and_level`.

- [ ] **Step 5: Write unit tests verifying roundtrips**
  Add roundtrip tests in `src/save.rs` for player data, level data, and chunk data serialization, verifying compression ratio and correct recovery of values.

- [ ] **Step 6: Run automated tests**
  Run: `cargo test`

### Task 4: Integrate Load/Save Flow in State

**Files:**
- Modify: `src/state.rs`

- [ ] **Step 1: Initialize SaveManager in State::new**
  In `State::new()`, initialize the `SaveManager`. Check if world files exist:
  - If yes, load level time and seed, restore player position, health, hunger, inventory, and camera position.
  - If no, keep default spawning values.

- [ ] **Step 2: Hook up chunk loading from saves**
  In `State::update_chunks()`, instead of always calling `Chunk::new(cx, cz)` to generate terrain:
  - Attempt to load chunk data from `SaveManager::load_chunk`.
  - If loaded successfully, deserialize block arrays, fluid levels, light levels, and recalculate heightmap.
  - If not found, fall back to procedural generation.

- [ ] **Step 3: Hook up chunk saving on unload**
  In `State::update_chunks()`, when a chunk is unloaded because it falls outside the render distance, extract its block and light arrays and save them.

- [ ] **Step 4: Verify compilation**
  Run: `cargo check`

---

### Task 5: Implement Thread-Based Autosaving

**Files:**
- Modify: `src/state.rs`

- [ ] **Step 1: Setup background worker thread**
  In `State::new()`, spawn a background worker thread that listens on a channel for `SaveCommand`s and writes region and metadata files sequentially.
  Store the channel sender on `State`.

- [ ] **Step 2: Add autosave timer**
  In `State::update(dt)`, accumulate elapsed time. Every 300 seconds, send level data, player data, and current dirty chunks to the worker channel, and reset the timer.

- [ ] **Step 3: Verify compilation**
  Run: `cargo check`

---

### Task 6: Hook Up Shutdown Flow & Save Screen UI

**Files:**
- Modify: `src/state.rs`
- Modify: `src/app.rs`

- [ ] **Step 1: Add Save Screen state and render overlay**
  - Add `is_saving: bool` to `State`.
  - In `State::render()`, if `is_saving` is true, render a dark translucent screen overlay displaying `SAVING WORLD...` centered on screen.

- [ ] **Step 2: Update Pause Menu quit action**
  - Change Pause Menu button label from `"QUIT"` to `"SAVE AND QUIT"`.
  - In `handle_menu_click()`, when the quit button is clicked:
    - Set `self.is_saving = true`.
    - Perform a manual frame render to present the "SAVING WORLD..." screen.
    - Perform a synchronous save of all loaded dirty chunks, player state, and level metadata, joining/awaiting any pending background saves.
    - Exit the event loop.

- [ ] **Step 3: Update window CloseRequested handler**
  In `src/app.rs` under `WindowEvent::CloseRequested`, invoke the same synchronous save before calling `event_loop.exit()`.

- [ ] **Step 4: Run compilation check**
  Run: `cargo check --release`
