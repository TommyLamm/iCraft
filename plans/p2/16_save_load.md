## 16.1 世界序列化

### 新增/修改文件
- **[NEW]** `src/save.rs` — 存檔/讀取系統
- **[MODIFY]** `Cargo.toml` — 新增 `serde`, `bincode` 或 `flate2`
- **[MODIFY]** `src/state.rs` — 退出時保存, 啟動時加載

### 存檔目錄結構
```
saves/
├── world_001/
│   ├── level.dat          # 世界基本信息 (種子, 時間, 玩家位置)
│   ├── player.dat         # 玩家數據 (背包, 生命, 位置)
│   └── regions/
│       ├── r.0.0.bin      # 區域文件 (32×32 Chunks)
│       ├── r.0.1.bin
│       └── ...
```

### 子任務清單
- [ ] 定義存檔格式 (binary + 壓縮)
- [ ] `WorldSave::save(path)` — 序列化所有 Chunk 到磁碟
- [ ] `WorldSave::load(path)` — 反序列化恢復世界
- [ ] 區域文件: 每個文件包含 32×32 個 Chunk (類似 Anvil 格式)
- [ ] Chunk 數據壓縮: 使用 Palette 壓縮 + zlib/lz4
- [ ] 玩家數據序列化: 位置, 背包, 生命, 飢餓, 遊戲模式
- [ ] 世界元數據: 種子碼, 遊戲時間, 難度
